# 克隆实例
<NLevelAnalytics />

克隆实例功能允许用户在不同设备上创建应用的附加实例，实现数据同步和在多个设备上的无缝访问。

## 它是如何工作的？

1. **PeerJS 和 PeerJS 服务器：** 我们利用 PeerJS 技术和默认的 PeerJS 服务器为应用建立全面的数据连接。

2. **克隆实例的目的：** 克隆实例的设计与其他聊天应用提供的常见功能相吻合，允许用户在多个设备上访问其数据和聊天对话。

3. **注册流程：** 用户可以通过分享应用提供的链接或扫描二维码来初始化克隆实例。在确认同步之前，考虑添加两步验证以进行额外的验证。

4. **认证和安全性：** 我们正在积极努力加强认证系统，通过实施公钥加密进行同伴认证。此增强将导致临时连接字符串以实现安全认证，这与创建克隆实例完美融合。

5. **数据同步：** 克隆实例可以实时同步数据，但如果同伴离线，数据将在下次连接到应用时进行同步。

6. **兼容性：** 克隆实例可以在任何支持的浏览器上创建。但请注意，由于我们仍处于概念验证阶段，当前浏览器对此功能的支持有限。

7. **多个克隆和存储：** 用户可以创建多个克隆实例，作为数据存储的冗余备份。我们正在探索将存储位置配置为仅在克隆实例上的可能性。

## 结论

克隆实例功能通过在多个设备之间实现无缝数据同步，增强了用户的便利性和可访问性。请注意，由于我们处于概念验证阶段，功能可能存在限制和持续的开发来改进功能。请期待未来的更新和增强功能。
